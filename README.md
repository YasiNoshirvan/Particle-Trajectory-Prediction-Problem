# Particle Trajectory Prediction for Resistive Silicon Detectors

This repository contains a Data Science Lab project focused on predicting the coordinates of particles passing through resistive silicon detectors using machine learning regression models.

The project builds a complete data science pipeline, from preprocessing and feature selection to model training, hyperparameter tuning, and performance evaluation.

---

## 📌 Project Overview

In particle physics experiments, detector pads record signal measurements generated by particles passing through the detector.  
The goal of this project is to predict the **(x, y) coordinates** of each particle event based on signal-related features extracted from detector pads.

The task is formulated as a **regression problem**, since the target variables are continuous spatial coordinates.

---

## 🎯 Objective

The main objectives of the project are:

- Build a machine learning pipeline for particle coordinate prediction.
- Analyze signal features recorded by detector pads.
- Perform preprocessing, normalization, and standardization.
- Apply feature selection using Random Forest feature importance.
- Compare multiple regression models.
- Tune the hyperparameters of the best-performing model.
- Evaluate model performance using Euclidean Distance.

---

## 🧪 Dataset

The dataset consists of particle detector events represented as tabular data.

### Development Dataset

- **385,500 rows**
- **92 columns**
- Includes known target coordinates: `x` and `y`
- Used for training and validation

### Evaluation Dataset

- **128,500 rows**
- **91 columns**
- Includes event IDs but does not include target coordinates
- Used for final prediction

The dataset contains no missing values or duplicated records.

---

## 📊 Features

Each event is described by signal measurements recorded by detector pads.

| Feature | Description |
|---|---|
| `pmax` | Magnitude of the positive peak of the signal |
| `negpmax` | Magnitude of the negative peak of the signal |
| `tmax` | Delay from a reference time |
| `area` | Area under the signal |
| `rms` | Root mean square value of the signal |

Feature importance analysis showed that `pmax` and `negpmax` were the most influential features, while `rms` contributed less to the prediction task.

---

## 🛠️ Methodology

The project follows a structured data science pipeline:

1. **Data Exploration**
   - Checked for missing values
   - Checked for duplicated rows
   - Analyzed feature structure

2. **Feature Selection**
   - Used `RandomForestRegressor` to compute impurity-based feature importance
   - Selected the top 60 most relevant features
   - Reduced dimensionality and removed noisy or less informative features

3. **Preprocessing**
   - Normalized numerical features
   - Standardized data using Z-score normalization
   - Prepared development and evaluation datasets for model training

4. **Model Training and Comparison**
   - Trained and evaluated multiple regression models
   - Compared models using Euclidean Distance between predicted and actual coordinates

5. **Hyperparameter Tuning**
   - Focused on tuning the MLP model
   - Tested different hidden-layer structures and maximum iteration values

## 🤖 Models Tested

| Model | Description |
|---|---|
| Multi-Output Regressor | Regression model for predicting multiple outputs |
| Decision Tree Regressor | Tree-based regression model |
| Random Forest Regressor | Ensemble model also used for feature importance |
| Multi-Layer Perceptron Regressor | Neural network model for non-linear regression |

---

## 🏆 Results

The best-performing model was the **Multi-Layer Perceptron Regressor (MLP)**.

| Model | Euclidean Distance |
|---|---:|
| Multi-Layer Perceptron | **4.114** |
| Random Forest | 4.948 |
| Decision Tree | 8.999 |
| Multi-Output Regressor | 17.420 |

The final MLP configuration was:

```python
hidden_layer_sizes = (300, 255)
max_iter = 400
activation = "relu"
verbose = True
```

The MLP model achieved the best performance due to its ability to model complex non-linear relationships between detector signal features and particle coordinates.

---

## 📈 Key Insights

- Feature selection significantly reduced dimensionality and improved model performance.
- `pmax` and `negpmax` features were the most important predictors.
- Removing less informative features helped reduce noise and overfitting risk.
- MLP outperformed tree-based and linear multi-output regression models.
- Hyperparameter tuning had a strong impact on neural network performance.
